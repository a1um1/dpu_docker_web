Git

Git เป็นระบบควบคุมเวอร์ชันแบบกระจาย (distributed VCS) ที่ใช้จัดการการเปลี่ยนแปลงของซอร์สโค้ด ทำให้ทีมทำงานร่วมกันได้อย่างมีประสิทธิภาพ และสามารถย้อนกลับไปยังสถานะก่อนหน้าได้เมื่อจำเป็น

พื้นฐานสำคัญ

  • Repository (repo): โฟลเดอร์ที่เก็บประวัติการเปลี่ยนแปลง
  • Working Directory: ไฟล์ที่แก้ไขอยู่บนเครื่องคุณ
  • Staging (Index): พื้นที่เตรียมไฟล์ก่อน commit
  • Commit: บันทึกชุดการเปลี่ยนแปลงพร้อมข้อความอธิบาย
  • Remote: ที่เก็บระยะไกล (เช่น GitHub, GitLab)

ถ้าต้องการแทรกรูปภาพแผนผัง Git หรือ workflow ให้ใช้ตำแหน่งรูปด้านล่าง:

การใช้งาน GIT

คำสั่งพื้นฐาน (ตัวอย่าง)

git init                  # สร้าง repo ใหม่
git clone <url>           # คัดลอก repo
git status                # ดูสถานะไฟล์
git add <file>            # เพิ่มไฟล์ไปที่ staging
git commit -m "message"   # บันทึกการเปลี่ยนแปลง
git log                   # ดูประวัติ commit
git branch                # ดูสาขา
git checkout -b feature   # สร้างและสลับไปที่สาขาใหม่
git merge main            # รวมสาขา
git pull                  # ดึงและรวมจาก remote
git push                  # ส่งขึ้น remote

แนวปฏิบัติที่แนะนำ

  • ใช้ branch สำหรับฟีเจอร์หรือบั๊กแต่ละอย่าง
  • เขียนข้อความ commit ให้สั้นชัดเจนและมีความหมาย
  • รีวิวโค้ดผ่าน pull/merge requests ก่อนรวมเข้าหลัก
  • ใช้ `.gitignore` เพื่อไม่เก็บไฟล์ชั่วคราวหรือความลับ
